科研思索录（一）

发表于 2018-05-08

从我开始接触神经网络的时候，我就在思考这类方法到底为什么能取得如此如此成就。它几乎颠覆了所有的领域，曾经有一段时间只要是将自己的方法与神经网络结合起来就能提升性能。但同时这又是一个挺起来有点玄学的方法，因为这类方法对参数的敏感性，调参的过程被戏称为“炼丹”。

而我并不想靠一时的脑洞去发现新方法，也不想刻意炼丹去刷 state-of-the-art ，我更希望能明白我们做研究或者工程面临的问题有哪些，可以从哪些方面入手改进。正如一位院士所说，“把研究建立在严谨的理论、模型建造、实验验证和统计分析的基础上”。

从一个比较宏观的层面去思考这个领域，我发现有诸多问题尚未得到解决。下面列出了我思考过的一些问题：

  • 很多时候模型的改进都是基于某种经验／直觉。这样的直觉都有哪些？有没有办法做数学化的陈述？
  • 神经网络模型是如何对数据空间进行泛化的？这样的泛化先验是否能够显示地表达？
  • 在高维空间下的数据采样对机器学习算法有怎样的影响？错误的数据对模型又有怎样的影响？
  • 某些特定类型的数据（图像／声音／文本）存在着怎样的分布／先验？
  • 神经网络什么时候会犯错？错例对模型本身的 debug 有没有帮助？
  • 为什么会出现对抗样本？如何提高模型的鲁棒性？
  • 神经网络学到的知识是否能够解释？神经网络学到的知识能否用于指导人类？

目前对我来说，这些问题很难找到非常本质的回答，于是我想写一系列文章讲述我收集到的资料和一些思考。

为什么要做这件事呢？因为现实世界中的问题是很复杂的。唯有逐步理清思路，才会知道自己到底在做什么，自己的工作能否推动科学前进一小步。

举个例子，很多时候我们在求解一个很高维的问题。这个时候一些在低维空间中的一些直觉就不成立了。这个被称为维度灾难（Curse of Dimensionality）。

考虑一个半径为 1 的正立方体，体积也是1。在高维空间中，半径为 1 的 $K$ 维正立方体，体积同样是 1。假如有一个正立方体体积和 1 非常接近，$r = 1-\epsilon$, 它的体积从直觉上说是和1差不多的。比如当 $\epsilon = 0.01$ 时，体积为 $(1-\epsilon)^3$ = 0.97。但是维度上升到120维的时候，体积就只有 0.3 了；维度上升到320维的时候，体积就只有 0.04 了；维度上升到1000维的时候，体积就只有 $4.3 \times 10^{-5}$ 了。高维空间里面可谓是失之毫厘，差之千里。

CV 领域最入门的数据集 mnist 是 28x28 像素的，有 784 维；最简单的彩色数据集 cifar-10 是 32x32 像素的，三个通道共 3072 维。在许多网络结构在最后几层的全连接当中，有一两千维的特征是很常见的，这些都是高维空间的数据处理。如果对维度没有直觉，有时就难以理解实验中遇到的一些现象。

最近在做的研究也涉及到对一些本质问题的思考，希望能坚持讲这个系列写下去，能做出一些有突破性的成果吧。

一些思考

发表于 2018-01-03

对抗样本出现的原因到底是怎样的？许多观点认为跟他是线性的有关，也有观点认为决策边界太奇怪导致的。（TODO：分析输入图像在被扰动之后，信息在不同的层发生了某种程度的放大导致最终出错。）神经网络出错的过程可能类似我们玩的一个游戏 “道听途说”，第一个人看到一个词语，然后每次前面的人通过某种方式将信息传给后面的人，要求最后一个人说出答案。在这个过程中一旦某一层出了偏差，整个过程都会出问题。

当然我没有做过实验，不知道信息在哪一层出线了偏差，但是如果每一层都保证足够稳定应该就没有问题了。下面用数学符号来表示这个想法。

图像是一个向量 $x \in R^{n\times m}$，神经网络是一个多层函数：$f(x) = f_K(f_{K-1}(…f_1(x; W_1)… W_{k-1});W_K)$

定义图像的邻域 $N_F(x) = \{x’ |x’ \in R^{n\times m}, \left|Q\right|_F < \epsilon \}$ 根据范式的不同，邻域可以包含 $N_0, N_2, N_{\infty}$

数学上有个 Lipschitz条件，存在一个常数L使得 $|f(x)-f(y)| < L|x-y|$。我们用的卷积层／全连接层都是线性变换。对于一个线性变换，可以推导出 $|f(x) - f(y)| < \lambda_{\max} |x-y|$。如果我们用一些方法减小这个 $\lambda_{\max}$ 就可以比较容易达到 $L_2$ 稳定了。

类似的，如果能找到一个数学范式，使得 $\max_{y\in D(x)} \left| f(x)-f(y) \right|< \epsilon$ 也能保证其稳定。那么现在的一些改进，比如直接学习频率域的滤波器，是不是可以在这方面得到一个解释？

还有一个野路子：对图像作不同程度的模糊（低通滤波），然后将不同类型的图像分别训练（或者一起）分类器。输出的时候将网络图像不同模糊级别的结果综合一下。这样maybe可以应对一部分？

或者也可以这么做：训练一个网络，然后用某种方式生成对抗样本，再用对抗样本继续训练，观察网络内部发生了怎样的变化。将这种规律找出来，作为先验，用于改进网络。很早就有人提出过这样的说法，训练好的深层神经网络是对训练数据空间的泛化先验。深层网络通过学习训练数据使得训练数据的识别能够完美拟合，同时它还能保证训练数据附近的区域也能够识别正确。如果我们强制约束“输入在邻域内稳定”这个先验，得到的神经网络可能会更加鲁棒。

动态规划的技巧（一）：换元法

发表于 2017-04-10

动态规划问题（DP）是程序设计竞赛中十分有趣的一类问题。

DP有很多技巧，笔者积累了一些常见的思考技巧，总结成了系列。在解题遇到困难的时候不妨试试这些方法。

动态规划问题的一般形态

读到这篇文章我相信大家已经了解了最优子结构、无后效性这些概念，这里不再赘述。

求解动态规划问题的时候一般会得到一个转移公式：

$$f(s_d) = opt [ f(S_i) + cost(s_i, d) ], s_i \in S $$

其中$S$是所有能够转移到$S_d$的状态的集合, $opt$ 表示最优，一般是max或者min。例如树形结构$s_d$ 表示一个树，$S$表示其所有直接子树。函数$f(\cdot )$用于优化某个目标值。很多时候状态可以由一个或多个维度的数表示。例如背包里面用体积表示状态，在例如最长公共子序列(LCS)问题可以用两个序列中的位置对表示。

考虑不同维度变量的关系

我们以二维的状态$(i, j)$为例，假设$f(i, j)$的值为$k$，此时固定了$i$和$j$去优化函数值$k$。那么不妨考虑如果将函数值$k$放入状态中，原来的状态$i$和$j$是否可以最优化。

我们以最长公共子序列为例子，两个数组记为$A$和$B$。确定A的前缀i和B的前缀j，可以求解最长上升子序列长度$k$。如果确定了i和k，那么j肯定是越小越好。此时可以列出方程：
$$f(i, k) = \min[ f(i-1, k), j | j > f(i-1, k-1) and B_j = A_i ]$$
结合数据结构可以求解此方程。

这个想法很简单，但是很多时候很有用。那么我们接下来看几个例子：

最长上升子序列

在最长上升子序列问题通常的求解方法中，我们能得到一个方程 $f(i) = \max( f(j) + 1 | a_j < a_i ) $。枚举了前$i$个数之后，考虑最长上升子序列的值确定的时候，自然是要找到一个位置$j$使得$a_j$最小。
我们用$v_i$记录最长上升子序列的长度为$i$的时候，最小的$a_i$值为多少。（假设初始值为正无穷）通过观察可以发现，添加一个数的时候，只可能会改变v中第一个大于或等于这个数的值。这样就得到了一个$O(n log ans)$的做法。注意这里的复杂度是和答案大小有关的，而答案大小是不会超过$n$的。

例题：圣诞礼物

这是我很久前出的一道题，题目大意是有N个物品，每个物品有一维或者二维的属性；每次可以从左边或者从右边选择一个装进袋子里面。袋子有个限制，就是每一个维度的属性和都不能超过某个上限。问至少需要多少个袋子。

这道题可以很显然的写出一个$O(N^4)$的DP方程：
$$f(l, r) = \min( f(l - i, r + j) | i \leq 0, j \geq 0, sum(l-i, l) + sum(r, r+j) \leq Limit )$$

f(l, r)表示从外往内左右分别取到了l, r，最少用多少袋子。

可以通过指针扫描的方法将其优化为$(N^3)$。现在我们用换元法这个技巧来试试，假设用了k个袋子，那么确定$l$之后$r$越小越好。此时可以得到新的方程。

如果$x=2$ and $y=2$，而且初始状态$a+b\geq 3$也是无穷的
其他情况模拟即可。

DFS的时候贪心。考虑连接(1, u)这条边，从1到u上的节点分别为v0=1, v1, v2…, vm。
那么从v0到vm/2的所有点距离不变，但是v[m/2+1]到vm所有点距离减少了。设ci是经过vi其他分支的节点数。
那么减少的代价是：
$$\sum_{i=m/2+1}^{m} c_i * (2i-1) $$
所以与处理一下前缀和即可。

C

与处理每年的9之和，然后枚举第一年的每一天和最后一年的每一天。

D

动态规划：$f[i][j]$表示有i个独立集的森林（包含空集），去掉所有树的根之后剩下的森林有j个独立集 所需要的最少的点数
$$f[i][j] =\min( f[j][i-j]+1, \min_{x|i,y|j}(f[i/x][j/y] + f[x][y]) )$$
直接$O(n^2log^2n)$枚举即可。

E

很多搞法，你可以写一个C程序然后反汇编。。
一个简单的做法，递推算出斐波那契数列，然后读入n，将stack[n]copy到最后面。不需要sub, jmp, cmp 操作。

F

贪心，扫描线。每次将右端点最远的区间删去。

暴力搜索可以过。
另外，你可以将这些列取出来，然后排序，判断是否相等即可。

H

答案就是每个点成为saddle point的情况数之和，这样就之和一行一列有关了。
预处理一下每个点至少删多少数才能称为saddle point，然后剩下p个的话，对答案的贡献就是2^p。

基于概率算法的求解思路

发表于 2017-03-28

在求解问题的时候，我们通常会使用一些并不是完全正确，但是出错的概率及其小的方法。例如在用哈希方法求解字符串相关问题的时候，有些时候我们会假设散列值相等的字符串一定相同。虽然也会出现冲突但是总体来说只要算法得出正确结果的概率足够大，我们可以认为它是可行的。

现实中这样的算法很多，例如之前提到的字符串哈希，还有素数判定 Miller Rabin算法。这里我们将介绍几个用此方法求解的例子，希望启发大家思考。

素数测试 Miller Rabin算法

Miller-rabin算法是一个用来快速判断一个正整数是否为素数的算法。它利用了费马小定理，即：如果$p$是质数，且$a, p$互质，那么$a^{(p-1)}mod\ p$恒等于1。也就是对于所有小于p的正整数a来说都应该复合a^(p-1) mod p恒等于1。那么根据逆否命题，对于一个p，我们只要举出一个$a, (a \lt p)$ 不符合这个恒等式，则可判定p不是素数。

Miller-rabin算法就是多次用不同的$a$来尝试$p$是否为素数。当有多个满足关系的$a$时，$p$为素数的概率趋向于1。试试上，Rabin在其论文中证明了进行了$k$次测试的错误率在$4^{-k}$以内。当我们做50次测试的时候，理论上误判的概率约为$7.89e^{-31}$。

判定边集是否是割集

对于一个连通的无向图，快速判定一个边集合是否存在割集，其中边集大小不超过15。

有另一种哈希的方法：构造出一棵树，给每一个非树边随机一个权值，然后定义树边的权值为所有能覆盖到它的非树边的权值的异或和。如果这个边集包含一个割集，那么一定存在一个边的子集，其异或和为零。我们将这个定理反过来用，通过判定是否存在这样的子集来判断边集是否存在割集。

我们来分析下出错的概率：假如边权用一个64位整数存储，$f(n)$表示$n$条边的集合中不存在异或和为零的子集的概率。很容易得出
$$f(1) = 2^{-64}$$
$$f(n) = (1 - 2^{-(65-n)}) \cdot f(n-1) (n > 1)$$
当n = 15时，不出错的概率约为 0.9999999999999983。

某一类组合游戏的应用

在组合游戏中，我们可以通过一些递推来得知某些状态是必胜还是必败。例如经典的取石子问题，假设一堆石子有n个，每个人每次都能取$v$个（$v$属于某个集合S）。先手面对这样的状态，可以让后手面临有$n-v | v\in S$个石子的状态，而只要其中一个状态会后手必败，则先手必胜；否则先手必败。

定义f(n)=0/1表示是否必胜，这个推导过程可以得出方程：
$$f(n) = 1 - \min{f(n-v) | v \in S}$$

观察这个式子，我们发现递推的过程中只要右边存在一个值等于1，就可以不用计算其他值了。当$S$这个集合较大的时候，我们可以使用一种随机算法，每次随机选取$S$的一个子集进行求解。此算法在某些数据中十分有效，而在某些数据中效果一般。

输出一个合法方案的问题

考虑这么一个问题：给定N个字符串，你需要选出其中的若干个，按照其原先的相对顺序连接在一起。求长度恰好为K且字典序最小的字符串。（1≦N≦2000，1≦K≦10^ 4，字符串长度之和不超过10^6）

这道题可以通过动态规划预处理，得到一个暴力的做法：如果不考虑字典序问题，我们可以通过动态规划求出某个字符是否可能出现在生成的字符串的第$i$位，将这些位置的集合定义为$Pos(i)$。很明显$Pos(i)$可以由$Pos(i-1)$转移得到。

那么我们只需要每次在$Pos(i)$中保留最小的字符，然后令它们转移到$Pos(i+1)$就可以求出答案了。此时我们发现转移的复杂度和$Pos(i)$的大小相关。但同时，我们发现最优答案不止一个，随机舍弃一些位置不一定影响答案。根据这个思想，转移的时候当$Pos(i)$太大，则随机舍弃一些位置，使得$Pos(i)$的大小在某个数$R$内，我们就可以在$O(R)$的时间内完成转移了。

小结

以上几个例子分别可以抽象成这么几种情形：

  • 以某种方式赋予哈希值。当两个元素哈希值不相等时，很大概率认为这两个元素不相同。
  • 有一个定理 $p \rightarrow q$，其逆反命题$\neg q \rightarrow \neg p$。当找到一个$q$不成立的时候可判定$p$不成立，当多次尝试下$q$成立，那么我们认为$p$有很大概率成立。
  • 当问题有很多最优情况的时候，随机舍弃一些情况也很可能达到最优。

当然这些方法也是有风险的。如果你没有初始化随机种子，命题人可能出到刚好卡掉你的数据；如果别人知道你的方法，也可以构造一些数据进行hack。例如有种网络攻击叫HASH COLLISION DOS，利用后台基于散列表实现的漏洞，构造特殊数据使得后台低效。
